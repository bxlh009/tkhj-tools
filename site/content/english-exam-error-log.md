A practice score tells you what happened once. An error log should change what you do next. If it records only the question number and correct answer, it becomes an archive rather than a study system.

## Record decisions, not embarrassment

Use one row for every wrong answer and every correct answer you were uncertain about.

| Field | What to write |
| --- | --- |
| Task | The exact question or writing skill |
| My decision | What I chose or wrote |
| Evidence | The text, rule, or descriptor that controls the answer |
| Error type | The reason the decision failed |
| Repair rule | One action for the next attempt |
| Review date | When you will test the rule again |

Keep the language neutral. “Careless” does not explain a decision. “Ignored the word *only* and accepted a broader claim” does.

## Use a small error taxonomy

Start with six labels:

1. misunderstood the task;
2. could not locate evidence;
3. misread a limiter or relationship;
4. used outside knowledge;
5. knew the rule but executed it incorrectly;
6. ran out of time.

Add a new label only when the existing labels genuinely cannot describe the error. Too many categories hide patterns.

## Write a repair rule you can observe

A strong repair rule begins with an action and a trigger.

Weak: “Improve vocabulary.”  
Stronger: “When a target word is unfamiliar, mark its grammatical role and the nearest contrast word before checking options.”

Weak: “Manage time better.”  
Stronger: “If I reread the same evidence twice without a new conclusion, mark the item and continue.”

The rule should fit on one line and be usable during the next practice.

## Separate knowledge gaps from process gaps

A knowledge gap means you lacked a word, grammar rule, or test-format fact. A process gap means you had enough information but used it poorly.

Treat them differently:

- knowledge gap: learn the item, create one example, then retrieve it later;
- process gap: repeat a similar task while following a visible checklist;
- time gap: rehearse a stop rule under a shorter limit;
- attention gap: identify the missed signal and highlight that signal in the next set.

Buying a larger vocabulary list will not fix a habit of ignoring contrast. Repeating full tests will not fix a specific article error as efficiently as a focused sentence drill.

## Schedule retrieval

Review an error after a delay rather than rereading the explanation immediately. A simple schedule is one day, one week, and three weeks.

At each review:

1. hide the old answer;
2. solve the original or a parallel item;
3. explain the controlling evidence;
4. mark the rule as retained, weak, or still failing.

If the same rule fails twice, make the repair action smaller. “Map the paragraph” may need to become “write a three-word job beside each paragraph.”

## Turn weekly data into a plan

At the end of the week, count error types rather than total mistakes. Choose:

- one high-frequency process error;
- one important knowledge gap;
- one timed recovery skill.

Build the next week around those three items. Keep other skills in light maintenance. This prevents a single poor practice score from causing a complete and unnecessary plan change.

## Example entry

**Task:** IELTS True/False/Not Given  
**My decision:** False  
**Evidence:** The passage mentions an increase but gives no cause.  
**Error type:** Treated missing evidence as contradiction.  
**Repair rule:** Choose False only when I can state the opposite claim from the text.  
**Review date:** Tomorrow, using three new cause statements.

This entry is useful because it names a transferable decision boundary.

## Keep the log honest and small

Archive rules that you have applied correctly three times across different material. Keep recurring rules visible at the top of the page. The active log should fit on one screen or sheet so it remains a tool you actually use.

The goal is not to eliminate every wrong answer. It is to stop making the same decision error without noticing.
