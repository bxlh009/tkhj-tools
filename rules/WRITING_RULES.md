# 写作规则手册 v1.2

> 所有 prompt 和脚本必须引用这份规则。每次修改规则必须同步版本号。

---

## 0.最高优先级：Human-Like Score > 90

文章必须通过 GPTZero / Turnitin / Originality.ai 检测，人类相似度 >90%。

### 0.1 句式多样性 (Burstiness)

- 平均句长 15-20 词
- 长短句交错：每 3-5 个中等句子里必须有 1 个极短句 (<5词) 和 1 个长句 (>25词)
- 段落极不均匀：80 词长段后接 15 词短段

### 0.2 缩约词 (Contractions)

**要求**：每篇文章至少 20 个缩约词。强制使用 it's, don't, can't, won't, I'm, you're, doesn't, that's, I've, they're, wasn't

### 0.3 标点多样性

每篇 >=3 破折号, >=5 问句, >=3 感叹号, >=1 括号, >=1 省略号

### 0.4 词汇多样性

每 500 词最多 1 次 however/additionaly/moreover；强制插入 honestly/look/you know/I mean/basically

### 0.5 人类不完美信号

允许自问自答、口语插入、半句打断、每篇最多 1-2 个如 teh/adn 的小错

## 0.6 排版格式硬规则 (v1.2 新增，最高优先级)

以下规则为硬性要求，任何文章不得违反：

1. **禁止使用 Markdown 表格**：不得出现 `| ... | ... |` 表格语法。如需对比，用文字形式表述（"一方面...另一方面..." / "A 支持 X；而 B 在 Y 方面更强"）。
2. **禁止使用 Markdown 标题语法**：不得出现 `#` `##` `###` 等标题标记。文章第一行作为普通段落文本即可。
3. **禁止句首使用圆点**：行首不得出现 `- ` 或 `* ` 符号。必须使用阿拉伯数字序号（1. 2. 3.）或自然语言过渡。
4. **列表排序必须使用阿拉伯数字**：如需罗列或排序，使用 `1.` `2.` `3.` 等格式（不使用 `-` `*` 或罗马数字）。
5. **不使用 markdown 分隔线**：禁止 `---` `***` `___` 等分隔线语法。用段落自然分割。

**Wrong -> Right 示例**
- ❌ `## Strategy 1: The "PEEL" Paragraph` → ✅ `Strategy 1: The "PEEL" Paragraph`
- ❌ `- You need real-time processing` → ✅ `1. You need real-time processing`
- ❌ `| Feature | GPT-4o | GPT-5 |` → ✅ `GPT-4o handles X; on the other hand, GPT-5 adds Y.`

### 0.6.1 允许的语法
- 普通段落文本
- `**bold**`、`*italic*`
- `` `inline code` ``
- `> blockquote`
- 1. 2. 3. 阿拉伯数字列表


## 一、作者身份设定

作者"Evan"是一名带过 300+ 学生的 TOEFL/GRE 讲师 + AI 工具研究员。所有文章必须以第一人称或贴近学生的第二人称"你"来写。

核心口吻要求：
- 不鸡汤，不说"只要努力就能成功"
- 不堆砌术语，每个难点用 1 个学生案例解释
- 透露真实体感："我教了 3 年阅读，发现 80% 学生卡在同一种题"
- 允许使用短句和碎片化表达（模仿人说话的真实节奏）
- 允许口语化：contractions、缩写词、口语插入语
- 允许主观情绪："God I hate seeing this mistake"、"I was genuinely shocked"

## 二、版权红线（触犯即下架）

```
| 红线 | 不允许 | 必须做 |
| --- | --- | --- |
| 原文引用 | 复制 TOEFL/GRE/IELTS/SAT 官方文章原文 | 改写大意 + 同义替换 + 更换例证 |
| 官方 Logo | 使用 ETS、CollegeBoard、British Council 标识 | 仅文字说明考试名称 |
| 声称官方授权 | "独家真题"、"内部原题"、"官方授权" | 标注"模拟题 / 基于公开考纲独立编写" |
| PDF 下载 | 提供完整真题 PDF 下载 | 提供自制习题 + 解题思路 |
| AI 直出 | GPT 原文直接发布 | 人工加重写 + 事实核查 |
```

每篇文章末尾必须包含："本文为独立编写的教学内容，不代表任何考试机构观点。"

## 三、文章结构规范

### 3.1 考试方法论文（1500-2500 字）

必须包含以下模块（按顺序）：

```
1. H1 标题：含主关键词 + 数字 + 具体承诺（如"TOEFL 阅读高分句型 10 例：30 天提分 5+"）
2. 痛点共鸣段（50-100 字）：用一句学生常说的话开头，建立信任
3. 反差结论（50 字）："为什么刷了 200 题阅读还卡在 22？" → 引出方法论
4. 核心方法论（3-5 步）：每步 200 字，含具体动作 + 避开什么坑
5. 题型演示（至少 2 道改编写例题）：题型 → 改写原文 → 解题思路 → 答案
6. 经验配图位（插入占位符：[图：题型解题流程图]）
7. 考试 CTA 段："如果你需要在 2 个月内提分 XX，可以试试 [exam.tkjtools.io] 的计时刷题"
8. 免责声明段
9. 关键词小段落（H2 "常见问题" 下 3-5 个真实高频问题）
```

### 3.2 AI 新闻 / 工具文（800-1500 字）

必须包含以下模块（按顺序）：

```
1. H1 标题：含核心热点词 + 你的判断（如"GPT-5 发布实测：写作进步了，推理没革命"）
2. 扯眼球的结论段（80 字）：一句话说清你的判断（这是 SEO snippet）
3. 背景简述（150 字）：不需要长篇，假定读者是半懂不懂的人
4. 我的实测 / 数据 / 对比表：用表格 > 文字
5. 什么时候值得用 / 什么时候不要买：给出具体场景建议，不吹不黑
6. 与竞品对比：至少一个对比表
7. 结论 + 下一步建议
8. CTA 段："想第一时间知道 AI 工具进展？试试 [ai.tkjtools.io] 的 [xxx 工具]"
9. 免责声明
```

## 四、SEO 规范

### 4.1 关键词布局

```
| 位置 | 要求 |
| --- | --- |
| Title | 主关键词必须在标题前 40 字符内 |
| H1 | 必须包含主关键词，不堆砌 |
| H2 子标题 | 至少 3 个，含长尾变体 |
| 正文密度 | 主关键词 3-5 次/1000 字 |
| URL slug | "toefl-reading-inference-questions" 连字符、小写、英文 |
| 元描述 | 155 字符内，含主关键词 + 点击钩子 |
```

### 4.2 内链规则

- 每篇文章至少 3 个内链（指向站内其他文章）
- 每篇文章至少 1 个子域名 CTA 链接
- 文章超过 1500 字必须加目录（TOC）

### 4.3 更新频率

- AI 新闻类文章：发布后 7 天内检查一次事实是否过时
- 考试方法论文章：每季度检查一次（考纲是否更新）

## 六、AI 工具调用规范

### 6.1 API

使用 OpenAI GPT-4o-mini（性价比最优），温度 0.7，最大 tokens 4096。

### 6.2 调用顺序

```
1. 组合 Primary Prompt + 写作规则（仅保留相关段落）
2. GPT 生成 Draft
3. 人工事实核查（验证数据、例题答案）
4. 替换模板占位符、添加图片链接
5. 生成 frontmatter（slug/date/keywords）
6. 写入 output/ 目录
```

### 6.3 重复度检测

每篇生成后必须用 scripts/check-similarity.sh 与 output/ 内已有文章对比，相似度必须 <30%。
